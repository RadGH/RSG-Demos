I

Executive Summary

Twelve feature additions comprising the v1.0 milestone expansion

This update marks the first major feature expansion of Vagrant & Ash beyond the core game loop. Twelve distinct features were designed, implemented, and tested — spanning AI asset generation, UI systems, game mechanics, audio synthesis, and offline infrastructure. The build now ships 245 passing tests across 13 suites with a compressed payload of approximately 218 kB.

  • ✦ OpenAI Biome Backgrounds — 14 AI-generated atmospheric landscape backgrounds, one per biome, created with DALL-E 3. Thematic prompts designed for dark fantasy atmosphere.
  • ✦ PixelLab Race Portraits — 84 pixel art portraits (12 per race × 7 races) generated via PixelLab AI. Selectable in character creation.
  • ✦ Portrait Selector — Players choose their character portrait during creation. Portraits persist throughout the entire run across all screens.
  • ✦ Randomize Character Button — One-click random character generation drawing from a thematic name pool of 30 curated names.
  • ✦ Font Awesome Icons — ~49 SVG icons replacing emoji throughout the UI. Scales cleanly and respects CSS color theming across light and dark contexts.
  • ✦ Pause Menu — Escape key (desktop) or ☰ button (mobile) opens a pause overlay. Options to resume or return to Main Menu without progress loss.
  • ✦ Animated Logo SVG — New atmospheric logo with drifting ash particles and ember glow pulse animation. Replaces the previous placeholder title text.
  • ✦ Asset Review Dashboard — /demo-assets/ page listing all generated assets for quality control review before shipping.
  • ✦ Low-Wit Easter Egg — Party with Wit < 5 sees slower typewriter text. Wit ≤ 1 garbles the text into runes and symbols — a subtle permadeath flavor mechanic.
  • ✦ Biome Header Backgrounds — Each biome's journey header now displays its AI-generated background for an immersive regional atmosphere.
  • ✦ Web Audio Procedural Sounds — 15 synthesized sounds with no external asset files. Combat effects, victory fanfare, and town bells fully synthesized in-browser.
  • ✦ Service Worker & PWA — Offline-first cache-first service worker. Bundle split into 4 chunks: vendor, engine, data, and main for optimal load performance.

IV

Bugs & Known Issues

Defects discovered during development, potential risks, and resolved items

Bugs Found
Portrait 404 Fallback
If portraits are not yet generated, character creation shows broken image placeholders with no indication.
Mitigation: onError handler swaps to a letter-avatar fallback immediately. Mitigated
Pause Button Overlap
The floating ☰ button may overlap with the mute button or other fixed-position elements at certain intermediate screen sizes.
Status: Minor layout conflict. Z-index review needed. Minor
Logo SVG on Safari
SMIL animation support in inline SVGs is inconsistent across Safari versions. The logo animation may render without motion on older Safari.
Mitigation: Logo uses CSS @keyframes, not SMIL. Needs Safari device testing. Pending test
Potential Issues
Background Image Load Time
14 large PNG backgrounds (~1 MB each) are loaded lazily per biome. First visit to each biome may show a brief blank delay before the image appears.
Recommendation: Add loading="lazy" and show the biome's solid color as a CSS placeholder. Watch
PixelLab API Rate Limits
Generating 84 portraits sequentially in a single script run may hit rate limiting. Some portraits may be absent on the first generation pass.
Recommendation: Add retry logic with exponential backoff and resume from checkpoint. Watch
Wit Easter Egg + Skip
When garbled text is skipped via tap, the full garbled version appears instantly. This reveals garbled content, not the original — which may surprise some players.
Note: Intentional behavior by design, but warrants player feedback. By Design
Previously Fixed
Notification System
SHOW_NOTIFICATION dispatches were stored in state but never rendered to the player — notifications were silently dropped.
Fix: Fully visible via animated NotificationToast component. Fixed
Save Corruption Recovery
Single-slot saves could be corrupted on write failure with no recovery path, causing permanent data loss on interrupted writes.
Fix: Dual-slot save with backup rotation prevents data loss on any write failure. Fixed
Bundle Size Warning
Vite was emitting a bundle size warning on every build due to an oversized monolithic chunk. CI output was polluted.
Fix: Resolved by manualChunks config splitting data/engine/vendor. Fixed
V

Recommendations

Prioritized follow-up work to strengthen the v1.0 foundation

1
Background Lazy Loading
Add IntersectionObserver to load biome backgrounds only when the biome header enters the viewport. This eliminates the upfront bandwidth cost of loading all 14 backgrounds and improves first-biome time-to-interactive meaningfully. Pair with a solid biome color as an instant CSS placeholder.
Estimated: 2 hours Performance
2
Portrait Caching Strategy
Store the selected portrait URL in save data. Pre-cache the chosen portrait on game load so it renders instantly during the journey without a network round-trip or visible flicker. Particularly important for offline PWA sessions.
Estimated: 1 hour UX
3
Icon Accessibility Audit
All <Icon> components should carry a title prop for screen readers in any context where the icon replaces meaningful text content. Run an axe audit pass and annotate every icon usage that lacks an accessible label.
Estimated: 2 hours Accessibility
4
Animated Biome Background Transitions
Crossfade between biome background images when the player's biome changes. A CSS opacity transition on the background img element creates a filmic quality that reinforces the sense of travel and distance without any JavaScript orchestration.
Estimated: 1 hour Polish
5
Auto-Save on Pause Menu Open
Auto-save the run state whenever the pause menu is opened so that "Return to Main Menu" can be offered without the risk of losing progress. This is a 30-minute change that significantly reduces player anxiety around the pause menu, especially on mobile where OS interruptions are common.
Estimated: 30 minutes Quick Win
VI

Future Vision

Planned expansions charting the road ahead for Vagrant & Ash

Near-Term  —  v1.1
  • Companion Story Arcs
    Each party member develops a personal story arc with 3 stages, unlocked through journey milestones and player decisions.
  • 6 New Biomes
    Completing the full 12-biome world map, each with unique encounter tables, enemy types, and AI-generated backgrounds.
  • Crafting System
    30 recipes using biome-specific materials gathered during the journey. Crafting takes rest actions and modifies inventory weight.
  • 11 Distinct Endings
    Final outcomes determined by faction allegiance, artifact collection, and major decision points across the journey.
  • Route Planning Screen
    Before each journey leg, players view a rough map and choose a path — trading safety for speed or resources for risk.
Long-Term  —  v2.0
  • The Hollow Campaign
    A second full journey traversed westward in reverse — the same world recontextualized by what the first journey revealed.
  • The Crossroads Moment
    If two save files exist, the two parties briefly meet mid-journey — an optional story beat that acknowledges the player's history.
  • Community Seeds
    Shareable run seeds with a lightweight leaderboard tracking survival distance, party composition, and deaths per biome.
  • Ghost Messages
    Dark Souls-style messages left by other players at biome transitions — curated through upvote mechanics to surface the most resonant notes.
  • Companion Bond Matrix
    Relationship values (−5 to +5) tracked between each pair of party members, affecting combat synergy bonuses and dialogue options.
