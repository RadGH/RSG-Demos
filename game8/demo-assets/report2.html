I

Executive Overview

Where the game stands after Milestone 62

Vagrant & Ash has grown substantially across 62 milestones. The core loop — march, encounter, decide, survive — is functional and compelling. Assets are in strong shape: 84 race portraits, 14 biome backgrounds, a full icon library. Systems like the skill consolidation, SOS encounter weighting, camp persistence, and carry-over characters have been recently implemented.

This report focuses on what's cracking beneath the surface: bugs that slip through unnoticed, systems that exist in code but aren't surfaced in gameplay, UX friction that makes the experience feel rougher than it should, and encounter content that needs thinning or enrichment. The goal is to give the next development pass a clear, prioritized target list.

II

Bug Inventory

Confirmed issues and suspected defects by severity
B-001 Save meta reads stale journey.dayNumber instead of top-level dayNumber Critical
src/services/saveLoad.js — getSaveMeta() reads parsed.journey?.dayNumber but day number is stored at parsed.dayNumber (top level of state). The Continue Journey button shows Day 1 for all saves. Fix: change to parsed.dayNumber ?? parsed.journey?.dayNumber ?? 1.
B-002 Wit below 5 easter egg mentioned in About panel is not implemented Critical
The About panel states "Wit below 5 has… consequences." No such mechanic exists in any component or engine file. Either implement it (garbled/slow text on low-Wit characters) or remove the reference from the About copy to avoid player confusion.
B-003 Skill consolidation doesn't migrate existing saves — old skill keys orphaned Major
src/services/saveLoad.js — The migration function only handles v1→v2. Characters saved before the consolidation (M60) may carry combat_blade, herbalism, lore_history etc. skill entries. These silently do nothing. Add a v3 migration pass that renames old keys on load.
B-004 Combat screen has no retreat path when all party members are incapacitated Major
src/screens/Combat/CombatScreen.jsx — If all party members reach 0 HP simultaneously (possible with AoE damage), the game may enter a state where no one can act but the combat loop continues. The game-over watcher may not trigger mid-combat. Verify the useGameOverWatcher hook fires during active combat.
B-005 Journal overlay shows entries for dead characters with no visual distinction Major
src/screens/Journal/JournalOverlay.jsx — Dead characters' journal entries appear interleaved with survivors' entries but nothing distinguishes their authorship. A deceased character's voice should feel archival — greyed out, marked with a small epitaph badge, or shown in a separate "Fallen" section.
B-006 EncounterCooldownDots uses journey.daysSinceEncounter which may not exist Major
src/screens/Journey/JourneyScreen.jsx — The new EncounterCooldownDots component reads journey.daysSinceEncounter. This field may not be initialized in initialState.js or incremented by journeyReducer. If undefined, all dots show as unfilled. Verify the field is tracked and reset after each encounter.
B-007 CarryOver characters applied before party is fully initialized Major
src/screens/CharacterCreation/CharacterCreation.jsx — The useEffect that applies vna_carryover from sessionStorage fires on mount. If the party members array has fewer slots than the templates (e.g. templates for 4 but default party size is 2), some dispatches target non-existent character IDs silently. Add a guard: only apply templates up to partySize.
B-008 Unspent-points conversion notification is never shown if SHOW_NOTIFICATION action requires a duration Minor
src/screens/CharacterCreation/CharacterCreation.jsx — The dispatch of SHOW_NOTIFICATION for the bonus resources message may fire before the phase transition completes, causing the journey screen to never display it if the notification reducer clears on phase change. Test that the message actually surfaces.
B-009 Portrait URL padStart(2, '00') is a no-op — should be '0' Minor
Previously in JourneyScreen.jsx: String(n).padStart(2, '00') — the pad string should be a single '0'. Though portrait images are now removed from the party bar, the same bug may exist in other places that construct portrait URLs (EncounterScreen, MapOverlay character markers). Audit all portrait path construction.

Systems Audit

Code exists but mechanics aren't surfaced to the player
Several systems were scaffolded early and are referenced throughout the codebase but have no visible effect on gameplay. These are the highest-leverage opportunities: the infrastructure is already built.
System Status Location Gap / What's Missing
Factions & Reputation Scaffolded src/data/factions.js
state/initialState.js		 Faction data and world.factions state exist. No encounter branches check faction rep. No UI shows current standing. Faction events could gate certain town services or spawn Ironwatch patrols at high hostility.
Quest System Scaffolded src/data/quests.js
hooks/useQuestWatcher.js		 Quest definitions and a watcher hook exist but quests are never started, displayed, or completed in UI. The watcher likely fires on every state change and checks conditions that are never met. Integrate 2–3 quests as a proof of concept.
Escalation / Tension Partial src/screens/Journey/JourneyScreen.jsx world.escalation is tracked and a warning icon appears above 40. But escalation never triggers additional encounters, boss spawns, or story beats. The tension system should drive pacing — make players feel hunted as it rises.
Scenarios Scaffolded src/data/scenarios.js Scenario data is defined but not selectable from New Game Setup. Scenarios offer enormous replayability — even one active scenario (e.g. "Hunted: start with Ironwatch escalation at 50") would add meaningful variety.
Voice Lines Unused src/data/voiceLines.js File exists with character voice line data but nothing displays or plays these. Even text-only "voice lines" shown in the travel log during key moments (first death, first combat, entering a new biome) would add personality.
Bond System Partial src/engine/bondEngine.js Bond logic is implemented but no UI surfaces bond strength between characters. Bonds don't visibly affect morale, combat, or encounter choices. Add a small bond indicator to the party bar and one encounter branch that checks for a bond.
Weather Effects Display Only src/engine/travelEngine.js Weather is displayed in the biome header but has no mechanical effect. Rain should reduce fire/fuel efficiency. Storm should add a chance of lost progress. Heat should increase water consumption. These are 1-2 line additions to the travel engine.
Camp Scavenge Submenu Partial src/screens/Camp/CampScreen.jsx Scavenge was requested as a submenu: generic scavenge, biome-specific scavenge, and hunt (with combat chance). Currently it's a single flat action. A submenu with 3 options and biome-flavored results would make camp feel more contextual.
Victory / Far Shore Ending Incomplete src/screens/Ending/EndingScreen.jsx An EndingScreen component exists. It's unclear if the Far Shore biome triggers it correctly or if the journey simply loops. The victory screen should feel earned — show the journey stats, surviving characters, notable moments, and a farewell paragraph.
Condition Duration Display Partial src/screens/Journey/JourneyScreen.jsx Conditions like Bleeding, Infected Wound, Exhausted are tracked with duration, but the party bar only shows condition icons with no count. Players can't tell if a condition will resolve tomorrow or in five days. Show the remaining duration as a small number on the icon.
Starting Equipment Choice Missing src/screens/CharacterCreation/CharacterCreation.jsx Requested feature: character background selection should offer a starting equipment option (gear budget items). The architecture exists (gearBudgetBonus on some backgrounds) but no UI allows the player to spend their gear budget in character creation — they just get a Traveler's Knife and go.

Encounter Content Audit

Encounter distribution, quality gaps, and missing content

The encounter pool is large — well over 100 entries — but distribution is uneven and certain biomes are dramatically under-served. More critically, combat encounters are rare relative to the game's emphasis on combat as a meaningful mechanic.

⚠ High Priority
Combat encounter frequency is too low
Players can go 8–12 days without a fight. The combat system is the most mechanically rich part of the game and it rarely fires. Target: ~25% of encounters should lead to combat. Currently closer to 10–15%. Add 2–3 combat-likely encounters per biome, especially ambush/bandit archetypes.
⚠ High Priority
Late-game biomes feel thin
Mountains and Storm Coast — the final stretch before the Far Shore — have the fewest encounters. This is exactly where tension should be highest. Add 4–6 encounters to each: brutal weather challenges, desperate refugees, imperial remnants, things that signal the world is ending right now.
◆ Medium
Skill checks favor History and Nature heavily
After consolidation, history and nature checks appear most frequently. Skills like combat, persuasion, deception, and stealth have far fewer checks. Players who invest in social skills feel their build is irrelevant. Add persuasion/deception branches to at least 6 existing encounters.
◆ Medium
No encounters reference the Pale Commissar chain after Day 22
The Commissar storyline ends at commissar_second_meeting (day 15–22). If the player hasn't resolved it, nothing happens. Add a "final reckoning" encounter in the Ironmarch or Mountains where the Commissar makes a last move — even if the player avoided the earlier meetings.
· Low
Multiple encounters have identical "nothing happens" failure text
Several skill check failures resolve with "You fail" or equivalent generic text. These should each have flavored failure descriptions — a failed Stealth check in Greywood should feel different from a failed Nature check in the Salt Flats. 20–30 failures need unique text.
· Low
Water encounters are fewer than food encounters post-SOS
The SOS system correctly boosts tagged encounters, but the water_source tag pool is smaller than food_source. In dry biomes (Salt Flats, Ironmarch) water crisis should feel more present. Tag 3–4 additional encounters with water_source and add 2 new Salt Flats water-crisis encounters.
V

UX Friction Points

Where players are likely to feel confused, lost, or annoyed
🔢
Stat max of 16 creates wildly unbalanced characters without warning
The stat cap was raised from 8 to 16 but the point cost and balance assumptions haven't been revisited. A character with Wit 16 and everything else at 3 is mechanically overpowered for knowledge checks. Either rebalance costs (stats above 10 cost 2 points each) or add a soft warning when any single stat exceeds 12.
📍
EncounterCooldownDots shows "7 quiet marches" to players who don't know the system
New players will see dots and have no idea what they mean. The label "quiet marches left" is confusing — does it mean something good or bad? Replace with a more intuitive metaphor: a flame that grows brighter as encounters approach, or simply omit numbers and show a "tension" fill bar.
🎒
No onboarding for new players — the About panel is the only reference
The first time a player opens the game, they see a main menu with no tutorial hints. The About panel helps but requires the player to seek it out. Consider a small "first march" hint system: a single line of grey italic text above the March button on Day 1, e.g. "March to begin your journey east."
⚔️
Combat screen doesn't explain what stats govern attack vs. defense
Players allocate points to Edge, Grit, etc. but the combat screen shows no indication of which stat is active or how damage is calculated. A small "ATK: Edge+Combat" indicator on each character's combat card would make stat investment feel meaningful and visible.
🎲
Skill check outcomes don't show the roll
When an encounter presents a skill check, the player sees a success or failure but never sees the dice roll. Showing "Roll: 8 vs. TN 10 — Failed" makes the game feel fair and teaches players the system. Even a brief flash of the numbers before the outcome would help.
📖
Journal entries all have the same font and visual weight
Personal journal entries (campfire thoughts), auto-entries from encounters, and combat summaries all look identical in the journal overlay. Differentiate by type: personal entries in italic with a candle icon, encounter entries with a compass icon, combat entries with a blood-red accent. The journal should feel like a real document.
🗺️
Map overlay doesn't show current day or resources — purely decorative
Players open the map to understand their situation, but it only shows position. Add a small status panel inside the map overlay: current day, food/water remaining, morale, current biome. Make the map a true tactical overview rather than just a progress bar with flavor.
💀
Death notification is easy to miss during fast multi-choice encounter resolution
Character death fires a notification, but if the player is rapidly tapping through encounter choices, the death toast may appear and vanish before they register it. Death deserves a full-screen pause — a brief black card with the character's name and their epitaph — before continuing. Currently it's a toast the same size as "Found 1 food."
⚙️
Character creation has no explanation of background special abilities
Background abilities like "Road Blessing" and "Field Surgeon" are described in text but players don't know when or how they activate. Add an "activates when:" line to each ability description. Players shouldn't need to guess whether "Once per rest" means the camp action or the Rest button on the journey screen.
🌡️
Morale percentage is shown but its effects on gameplay are invisible
Players see "Morale: 73%" but have no feedback on what morale actually does. The code checks morale for encounter branches and combat bonuses, but this isn't communicated. Add a one-line tooltip to the morale display: "Affects combat, encounter choices, and skill check rolls."
🏘️
Town entry button appears and disappears inconsistently
The "Enter Town" button on the journey screen only appears when journey.nearTown is truthy. But players don't know when they're close to a town, since the "Town Nearby" badge was removed. Consider replacing the badge with a subtle town-icon in the direction indicator: "→ East · ⛺ Waypoint ahead"
📜
Past Runs history doesn't show which biome the run ended in contextually
The history panel shows biome IDs like "ashfields" rather than the full name, and "cause" fields are often null. The cause of death comes from char.deathRecord.cause which may not be set in all death paths. Audit KILL_CHARACTER dispatches to ensure a cause string is always provided.
🔤
Free skill tags in character creation show raw skill IDs not display names
CharacterForm.jsx — The free skills section renders skillId.replace(/_/g, ' '), turning sleight_of_hand into "sleight of hand" (lowercase). Should use the SKILLS display name: "Sleight of Hand 2". Minor but visible polish gap.
🔊
Sound system exists but no audio assets are loaded
The src/services/sound.js service and mute button exist, but no audio files are bundled. The mute button is visible on the title screen but has no functional effect for new players. Either add 2–3 ambient audio files (fire crackle, wind, combat impact) or hide the mute button until audio is implemented.

Quick Wins

High impact, low effort — implement in under a session
⚡ Fix Now
Fix getSaveMeta() day number
One-line fix in saveLoad.js. The Continue button showing "Day 1" for every save is the first thing a returning player sees — it immediately undermines trust in the save system.
⚡ Fix Now
Add save migration for consolidated skills
20 lines in saveLoad.js. Any player who had a save before M60 will silently have broken skill references. A v3 migration pass protects existing players and demonstrates save reliability.
▲ This Week
Show dice roll result on skill checks
In EncounterScreen or OutcomeDisplay, show "Rolled 7 vs. TN 9 — Failed" on skill check outcomes. Players immediately understand the system, feel the tension, and stop wondering if outcomes are random or unfair.
▲ This Week
Add condition duration numbers to party bar
One-line change in JourneyScreen.jsx — add cond.duration as a subscript on each condition icon. Players currently have no idea if they need to worry about a condition or if it resolves tomorrow.
◆ Soon
Add weather mechanical effects to travelEngine
3–5 lines per weather type in travelEngine.js. Storm: +1 exhaustion on march. Heat: +0.5 water consumed. Rain: morale +1. These tiny changes make the weather display feel purposeful and add a new layer of resource tension.
◆ Soon
Differentiate journal entry types visually
CSS-only change in JournalOverlay.module.css + a type indicator in the entry render. Personal entries in italic, encounter entries with an amber left border, combat entries with a blood-red border. The journal becomes a beautiful artifact instead of a plain list.

The foundations are solid. The encounter writing is strong, the visual identity is cohesive, and the core loop — march, decide, survive — is genuinely compelling even at this stage. The gap between where the game is and where it could be isn't a lack of vision; it's a list of specific, tractable items. Fix the day number in the save meta. Show the dice. Give weather teeth. Let the Commissar finish his arc. These aren't moonshots — they're the difference between a game that works and a game that stays with you.
