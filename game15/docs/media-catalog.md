# Media Catalog

Every media file in the project is cataloged. The site reads catalogs to render gallery pages.

## Per-folder catalog

Each `public/images/<category>/`, `public/sfx/`, `public/music/`, `public/video/` directory contains a `catalog.json`:

```json
{
  "category": "background",
  "items": [
    {
      "id": "bg_forest_01",
      "name": "Mystic Forest",
      "file": "bg_forest_01.png",
      "thumb": "_thumbs/bg_forest_01.png",
      "source": "openai",
      "model": "dall-e-3",
      "prompt": "Dense mystical forest at dusk, pixel art style, 16-bit palette, dark greens and purples",
      "reference": "moodboard/forest_ref_03.jpg",
      "date": "2026-01-15",
      "notes": "Used as level 2 backdrop"
    }
  ]
}
```

### Required fields

| Field | Required | Notes |
|-------|----------|-------|
| `id` | yes | Unique snake_case |
| `file` | yes | Filename relative to the folder containing this catalog |
| `thumb` | yes for images/video | `_thumbs/<file>` — auto-generated thumbnail |
| `source` | yes | `openai`, `pixellab`, `elevenlabs`, `manual`, `svg`, `royalty-free`, `other` |
| `prompt` | if generated | Exact prompt used |
| `date` | yes | ISO date added |

### Optional fields

| Field | Notes |
|-------|-------|
| `name` | Human-readable label for the gallery |
| `model` | e.g. `dall-e-3`, `eleven-labs-v2`, `gemini-2.0-flash-image` |
| `reference` | Path to a reference image used in the prompt |
| `notes` | Where it's used, variant info, etc. |
| `tags` | Array of free-form tags for filtering |
| `prev` | Previous-version filename if this is a re-gen |

## Per-media-type top-level catalog (optional)

If a game has many sub-folders under `images/`, it may also keep `public/assets/images.json` aggregating everything (used by `images.html`). The aggregator is built from per-folder catalogs by a small script; per-folder catalogs are the source of truth.

## Rules

- **Add the catalog entry in the same change as the file.** No batch updates after the fact.
- **Generated assets without prompts are bugs.** If you can't recover the prompt, write `"prompt": "(unknown — generated before catalog rule)"` so it's flagged in the gallery.
- **Renames update both the file and the catalog `id`/`file`.** Never let them drift.
- **Thumbnails are required for images and video.** Generated by the optimization pipeline; if a thumb is missing, the gallery shows a placeholder and the build prints a warning.

## Checking integrity

A pre-release script (recommended addition per game) walks each media folder and reports:

- Files in folder but not in `catalog.json` → orphan
- Entries in `catalog.json` but no file on disk → broken
- Entries missing required fields → incomplete
- Images without thumbs → needs optimization

Run it before every milestone release.
