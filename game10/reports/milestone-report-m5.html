What Was Built

Milestone Feature Breakdown

M1 — Engine
⚙
Engine Foundation
  • ◆Fixed-timestep loop decoupled from render — physics locked at 60 Hz, render uncapped for fluid motion
  • ◆Wang tile corner-matching: 47-variant tileset for seamless grass-to-dirt transitions
  • ◆Smooth lerp camera with dead-zone and edge dampening
  • ◆AABB tile collision with sub-pixel correction on all 4 faces
M2 — World
▷
World & Objects
  • ◆5 world object archetypes, each with sprite sheet, shadow, and interaction radius
  • ◆Painter's-algorithm Y-sort ensures correct depth for all sprites each frame
  • ◆Minimap: 150x120 scaled canvas, fog-of-war mask, live enemy + player dots
  • ◆Day/night alpha overlay driven by real wall-clock time (20-min cycle)
M3 — Combat
△
Combat Core
  • ◆Three distinct AI classes, each with unique stat profile and behavior tree depth
  • ◆Melee hitbox computed as arc sector; correct for rotation and weapon reach
  • ◆Object pool pattern for both projectiles (40) and particles (200) — zero GC pressure
  • ◆Damage numbers: white standard / yellow critical / red player — lifetime + drift animation
M4 — Loot
◆
Loot System
  • ◆Item taxonomy: 6 types x 25 base definitions — weapons, armor, rings, trinkets, consumables, materials
  • ◆Weighted RNG drop table with clamped legendary floor rate
  • ◆Affixes: additive stat rolls selected from affix pool, non-duplicate per item
  • ◆Ground items render with pulsing billboard glow in rarity color
M5 — Equipment
★
Equipment & Stats
  • ◆Paper doll: 9 slots (head, chest, legs, hands, feet, main-hand, off-hand, ring x2, neck)
  • ◆Stat sheet aggregates base + level bonuses + item base + all active affixes in one pass
  • ◆Level-up modal: 5 attribute points per level, spend in STR / DEX / INT / VIT
  • ◆Tooltip diff: green arrow up for improvements, red arrow down for downgrades vs currently worn
Core Systems
◼
Architecture Highlights
  • ◆Pure Vanilla JS + Canvas2D — no framework overhead, predictable render budget
  • ◆Component-entity model: loose coupling between renderer, physics, AI, and input modules
  • ◆Vite HMR for fast iteration during development
  • ◆57 KB gzipped bundle — loads in <1 s on 4G

Known Issues & Gaps

AETHERMOOR // ISSUE_LOG // v0.5.0
[M10] PLANNED Touch controls not yet implemented. Mobile input is stubbed. Scheduled for Milestone 10.
[M10] STUB Gamepad API binding exists in source but action map is not connected. Controller input returns no-ops.
[M09] PLANNED Only Zone 1 (Ember Hollow) is playable. Zone 2 (Duststone Pass) is data-defined but not loaded. ETA: Milestone 9.
[M06] STUB Skill trees are fully defined in data layer (36 nodes, 3 paths) but the UI panel is not wired. Tree nodes return passive bonuses of 0. ETA: Milestone 6.
[M10] PLANNED Save system absent. All progression resets on page reload. LocalStorage serializer scaffolded but not hooked into game state. ETA: Milestone 10.
[---] WARN Day/night overlay may exhibit single-frame luminance flicker on Firefox when tab regains focus due to requestAnimationFrame gap. Workaround: force redraw on visibilitychange.
[---] EDGE Wang tile lookup falls back to solid fill color when JSON variant key is absent (rare map edge case). Visual artifact is minor but logged for future audit pass.
