Milestone Breakdown
M7 Visual FX Pass ✓
  • Building collapse: 20–30 debris particles in fire/rubble colors + white flash frame on destruction
  • Tornado enhanced with third outer layer (scale 1.6), 3 orbiting debris silhouettes, sky-darkening cone (depth 15, radius 200, alpha 0.08)
  • Car damage states: smoke particles at damageHp<2, fire+explosion burst at damageHp≤0
  • Flood tile ripple animations: expanding circles radius 4→20, alpha 0.4→0, looping at 1200ms
  • Lightning strikes every 18–35s: screen flash + jagged 3-segment bolt + scorch mark, player stun within 80px, thunder audio cue
M8 Zombie Variants ✓
  • Runner: 2× speed (145px/s), red tint, 0.8× scale, 240px chase radius
  • Brute: 40px/s, purple tint, 1.5× scale, 3 HP, knocks player back 300px/s for 0.3s
  • Screamer: stationary, emits yellow expanding ring every 1.5s, alerts all zombies within 300px into force-chase for 8s
  • Zombie stomping mechanic: player sprinting (speed>200, stamina>20) kills or stuns zombies
  • Car kills zombies on contact when occupied; kill counter HUD (bottom-left)
M9 Power-ups ✓
  • Medkit (green cross): restores 1 HP, respawns after 60s
  • Flare (orange triangle): press F to throw, repels zombies in 200px radius for 8s
  • Windbreaker (blue oval): 50% tornado suction reduction for 20s
  • Inventory HUD slot at bottom center with item icon display
  • Windbreaker countdown timer shown during active effect
M10 Dynamic Weather Events ✓
  • Fog event: 4 screen-edge tunnel-vision panels (alpha 0.82, depth 49), lasts 15s, recurs every 45–75s
  • Electrical storm: 6 jagged lightning bolts over 10s at 1.5s intervals, player stun within 120px radius
  • Hailstorm: damage every 2s, white rain visual, recurs every 80–120s
  • _showWeatherBanner() reusable announcement system for all event types
  • Weather HUD icons: fog/storm/hail emoji row at top-right during active events
M11 Full Audio Pass ✓
  • Generative music: 3-layer system — 55Hz drone with LFO, [A2/G2/E2] pad cycle, bandpass rhythm bursts
  • Dynamic intensity mix: tornado proximity + zombie count + HP + hail → 0–1 float, smooth ramp via setTargetAtTime
  • Car engine: sawtooth oscillator 80→260Hz pitched proportionally to velocity magnitude
  • Footsteps: road click vs flooded splash variant, paced to sprint speed
  • Fog ambience, hailstorm patter, electrical bolt sfx added to weather events
  • All audio fully procedural — zero audio file dependencies

Lessons Learned
01
One-shot event binding bites. Using keyboard.once for repeatable actions (flare throw) creates silent failures. Prefer keyboard.on with explicit state guards.
02
Procedural audio pays dividends. Zero file dependencies = zero load time, zero asset pipeline. The generative music system cost two days and eliminated an entire audio asset category.
03
Scene size compounds quickly. Adding weather events, car logic, and audio to GameScene in the same block tripled its size. Extract subsystems before adding M12+ features.
04
Weather timing needs cooldown guards. Any event with a duration needs an "is active" flag checked before re-scheduling. Fog stacking proved this the hard way.
05
Enemy archetype design is high ROI. Three zombie variants with distinct behavior loops created more gameplay variety than all the weather effects combined, for less code.
06
Cleanup listeners on entity death. Scene-level update listeners attached by individual entities (Screamer) must be removed in the entity's destroy path, not just on scene shutdown.
