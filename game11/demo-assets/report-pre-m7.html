Team Feedback
A
Alex Chen
Game Developer

Really excited about M8 zombie variants — this is the missing layer of tactical pressure. Right now every zombie plays the same way and the player learns to ignore them. Runners will force movement, Brutes will demand positioning, and the Screamer is a fantastic psychological threat.

"The Screamer is my favorite design yet — it doesn't hurt you directly but it completely changes threat calculus. Players will waste stamina avoiding it."

One concern: M10 FOG — implementing radius fog of war in Phaser without a RenderTexture mask layer is genuinely awkward. We'd need a full-screen Graphics overlay recalculated each frame. It works, but it's around 2–4ms of CPU overhead. I'd prototype it as a standalone scene before committing it to M10. If it's too expensive on mid-range phones, cut the fog radius or replace with a screen tint.

For M11 MUSIC — I'd scope it to: one base drone (sawtooth + low-pass), one kick-pattern OscillatorNode at 80bpm, one ambient texture pad. Dynamically crossfade between calm/tense/frantic mixes using GainNode automation. Don't try full generative melody — that's a rabbit hole.

J
Jordan Rivera
Project Manager

My strong recommendation: do M7 before M8. Visual debt is player-facing — right now building destruction is a puff of smoke and flooding is a color change. Players notice this. Fix the visuals first so M8 gameplay improvements land on a polished base.

"If M7 ships before M8, the zombie variant launch will feel premium. If M8 ships on top of abrupt building collapses, reviews will say 'good ideas, janky execution.'"

On M10: I'm flagging this as an optional stretch goal. Fog of war is the highest complexity, highest risk item in the block. If M7–M9 + M11 ship clean, M10 as a bonus keeps the cycle on schedule. Don't sacrifice M11's audio pass for M10 fog.

For M9 POWER-UPS: scope tight. One spawn per map, fixed locations, no inventory, no stacking. The flare mechanic is great but the double-tap needs a fallback dedicated button for accessibility — mobile Safari double-tap has a 300ms delay that hasn't fully gone away on all devices.

Architecture Improvements
01
Event Bus — Decouple UI from Game State
Replace per-frame polling in HUD and overlay components with a lightweight EventEmitter. Game emits player:hp-changed, tornado:suction-entered, weather:event-start. UI listens and updates. Reduces update loop overhead and makes adding M10 weather banners trivial.
02
Phaser Tilemap Layer for Map Rendering
Current graphics-based map rendering draws every visible tile as a Graphics primitive. Migrate to a StaticTilemapLayer using a spritesheet atlas. GPU-batched, culls off-screen automatically, and opens the door for flood tile animations as a second layer without CPU cost.
03
Entity Component Pattern for Zombie Variants
Before M8, refactor zombie behavior into composable strategy objects: MovementStrategy, AttackStrategy, AlertStrategy. Each variant assembles differently. Prevents M8 from becoming a chain of if/else branches in a 400-line zombie class.
04
AudioManager Singleton with State Machine
Current Web Audio code is spread across scene init and update. Centralize in an AudioManager with states: SILENT → AMBIENT → TENSE → FRANTIC. M11 generative music slots cleanly into this without touching game logic. State transitions driven by the event bus from improvement #01.
05
Bundle Analysis + Asset Atlas Before M7
Bundle is at 1.54 MB pre-M7. Run vite-bundle-visualizer and identify the largest contributors. Generate a single texture atlas (TexturePacker or free-tier Shoebox) for all sprite assets — this alone typically cuts texture memory 30–40% and speeds up Phaser scene load. Set a 2 MB hard cap enforced in CI.
Brainstorm Wall — Ideas Sparked by M2–M6
🏃
Survivor Rescue Chains
Rescued survivors follow the player in a chain formation. More survivors = slower movement but higher score multiplier. Creates a risk/reward convoy mechanic.
GAMEPLAY
🗺️
Procedural Neighborhood Types
Tag map zones as Downtown, Suburbs, Industrial. Each zone spawns different building densities, zombie concentrations, and item frequencies. Seeds become more distinctive.
WORLD
📻
Radio as Hint System
Emergency radio broadcasts reference specific map features — "Shelter spotted near the east road junction." Broadcasts give players directional info without a full compass.
NARRATIVE
🌪️
Tornado Merge Event
After 150s, two tornadoes merge into one mega-tornado with double suction radius and 3× debris. Acts as the final-minute escalation before helicopter arrival.
ESCALATION
🚧
Barricade Placement
Pick up debris from destroyed buildings and place it as a temporary barricade. Slows zombies 60%, breaks after 3 hits. Gives player a defensive tool without weapons.
GAMEPLAY
🌑
Night Mode
A darkness overlay where only the player's immediate area is lit. Tornadoes are visible by lightning flashes only. Dramatically changes threat awareness and pacing.
MODE
🏆
Ghost Replay
Record player path data for best runs (position + action per 100ms). On next play, show a translucent ghost of the best run. Encourages replay and social comparison.
META
🔊
3D Positional Audio
Use Web Audio PannerNode to position tornado and zombie sounds in stereo space relative to the player. Hearing a tornado approach from the left is viscerally alarming.
AUDIO
💡
Power Grid Collapse
Street lights glow at game start. Tornadoes touching power poles trigger a grid collapse sequence — lights flicker, then all go dark except the helicopter beacon.
ATMOSPHERE
🐕
Dog Companion
A dog NPC that barks when zombies are within 150px off-screen, acting as an audio proximity radar. Can be "sent" to fetch an item once per run. Adds emotional stakes.
NARRATIVE
📊
Post-Run Heatmap
After death or escape, overlay a heatmap of where the player spent the most time and where they took damage. Teaches strategy passively and is shareable as an image.
UX
🌊
Flash Flood Surge
A one-time flood surge event at ~90s: water level rises for 5s, flooding more tiles temporarily and briefly slowing all non-car movement. Adds urgency without permanent consequence.
