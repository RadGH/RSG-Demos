  • First-play tutorial overlay — step-by-step control guide with dismiss

  • Flare — repels zombies in radius for 8 seconds when used

Known Issues

Bug Tracker — M20 Snapshot

All known issues at the time of the v1.0 release cut. Severity and category annotated for M21 triage.

High Architecture

GameScene.js has grown to 600+ lines and mixes rendering, physics, audio, and state logic. Subsystem extraction into WeatherSystem, CarSystem, ZombieSystem is overdue before M21.

Medium World

Building destruction index can shift when two buildings collapse in the same frame, causing incorrect tile lookups and potential phantom rubble tiles on subsequent frames.

Medium Weather

Fog edge panels can stack visually if a fog event fires before the previous fog event's clear animation completes, causing a permanently darkened viewport.

Medium Items

Flare keyboard listener was previously registered as `once`, which caused a second flare use to silently fail. Fixed to per-use, but double-activation edge cases remain on rapid re-pickup.

Low Cars

Car physics body is always axis-aligned due to Arcade physics constraints — car cannot rotate visually while its hitbox remains rectangular, causing slight collision inconsistencies near walls.

Low Rendering

Night mode darkness implemented with 4 screen-edge panels rather than a RenderTexture mask. Corner regions where panels meet can appear blocky on non-square viewports.

Low Zombies

Screamer zombie's alert ring graphic is not cleaned up when the Screamer dies mid-alert, leaving an orphaned ring animation on the canvas until the next frame clear.

Low Audio

Generative music pad layer tempo does not re-sync when audio intensity changes mid-note, causing a brief timing drift between the rhythm and pad layers for ~1 beat.

Low Ghost Replay

Ghost replay does not account for random seed — a ghost recorded on seed A will replay on seed B's different map layout, making the ghost appear to walk through buildings.

Low AI

Survivor chain followers use linear interpolation toward the preceding entity's position, causing visible bunching when the chain turns sharply around a corner.

Roadmap

What's Next — M21–M30

The v1.0 release is the foundation. The next block takes Stormtown into living-world territory.

M21

Dynamic Dialogue

Survivors have voiced lines and personality-driven reactions. Sam comments on tornado proximity; bonus survivors bicker in the chain. NPC mood system affects chain speed bonus.

M22

Tornado Merge

Two tornadoes that collide merge into a super-tornado with combined suction radius and doubled destruction. Rare — triggers a screen-wide warning event.

M23

Companion AI

Sam gains an active role — she can be directed to hold a position, distract zombies, or call out hidden power-ups nearby. Expands the rescue from escort to partnership.

M24

Challenge Mode

Weekly curated challenge scenarios — fixed seed, specific constraints (no sprinting, no items, night only), global leaderboard seeded from the daily challenge system.

M25

Visual Theme Selector

Alternate town palettes: Dustbowl (sepia), Midnight (full dark with neon signs), Arctic (snow tiles, frozen flood). Unlocked through meta-progression milestones.

M26

Performance Pass

Off-screen tile culling, particle pooling, canvas layering with dirty-rect tracking. Target: maintain 60fps on mid-range Android with 8 tornadoes and 24 zombies active.

M27

PWA & Offline Support

Service worker, manifest.json, install prompt. Stormtown runs fully offline after first load. Home screen icon. Splash screen on iOS.

M28

Credits & Lore

Scrolling credits sequence on first win, unlock lore fragments per achievement. Brief worldbuilding — why is Stormtown abandoned? What is Sam running from?

M29–M30

Stormtown 2.0 Planning

Architectural reset based on M20 lessons. Evaluate migrating to PixiJS for WebGL renderer. Plan multi-floor buildings, a second rescue target, and true co-op ghost mode.

Team Retrospective

Looking Back

Two perspectives from the team on what worked, what didn't, and what Stormtown taught them.

💻

Alex Chen

Lead Developer

Most Proud Of

The generative audio system (M11). Building a 3-layer ambient music engine from raw Web Audio oscillators — no assets, no samples — that dynamically intensifies as danger increases felt genuinely creative. It transformed the game's tension in a way no other feature did.

What I'd Do Differently

I let GameScene.js become a monolith. By M15 it was already too large, and every subsequent milestone made extraction harder. I should have enforced a 200-line file limit per module from M3 onward. The subsystem pattern (WeatherSystem, ZombieSystem) was obvious in hindsight and should have been the architecture from day one.

Favorite Emergent Mechanic

The tornado sacrifice. We built the chain mechanic (M13) expecting players to protect all followers. Instead, players immediately discovered that deliberately steering into a tornado's edge to lose the slowest follower was a valid escape strategy. No one designed that — it emerged from two systems colliding. That's the best kind of game design.

📋

Jordan Rivera

Project Manager

Scope Management Across 4 Blocks

Block 1 (M1–5) was disciplined — core first, systems second. Block 2 (M6–10) was where we started promising too much per milestone. M7 VFX and M10 Weather were both under-scoped on time estimates. Blocks 3 and 4 (M11–20) benefited from that hard lesson: smaller scope per milestone, more polish per feature.

Shipped vs Deferred

Multi-floor buildings, a second playable character, and sound settings were deferred across all 4 blocks. Each time they were deprioritized in favor of features with stronger gameplay impact. They're all planned for v2.0 — but honestly, the game didn't need them. The defers were correct calls.

Lessons on Agent-Based Development

The biggest surprise: agent parallelism compresses calendar time dramatically, but it does not compress integration time. Two agents building two systems in parallel still need a human review pass when those systems need to interact. The M8/M9 zombie variant + power-up integration required a full reconciliation session that wasn't budgeted for. Build that in every time.

Process

Agent-Driven Development

Stormtown was built end-to-end using AI agents in parallel. These are the honest findings after 20 milestones.

⚡

Parallel Build Speed

Running a frontend developer agent and a design agent simultaneously on independent subsystems (e.g. weather rendering + audio system) cut per-milestone wall-clock time by roughly 40% compared to serial development. The gains compound on milestone blocks with 3+ independent features.

🔍

Context Drift

The biggest agent failure mode observed was context drift — an agent assigned to M15 night mode did not have full context of M12 ghost replay, leading to a collision in the canvas rendering layer. Explicit handoff documents between milestones are mandatory, not optional.

🛠

QA Agent Integration

A dedicated QA agent running accessibility checks and mobile viewport validation after each milestone caught 12 bugs across the project that would have silently shipped. The iPhone 14 Pro viewport check alone blocked 4 layout regressions in M6–M10.

📚

Memory & Continuity

The memory/ directory pattern — future_milestones.md, MEMORY.md, CLAUDE.md — provided reliable cross-session continuity. Agents could bootstrap project understanding in one read pass. Without these files, each milestone session required 15–20 minutes of re-orientation.

⚖

Build Stability

Automated release scripts (release.sh, deploy_pages.sh) eliminated the manual build/copy/deploy cycle. Zero failed deployments across 20 milestones. The discipline of never manually touching the releases directory paid dividends every single sprint.

🎯

The Key Insight

Agent-driven development is not about replacing decisions — it's about removing latency between decision and execution. The highest-value moments were when a human provided a crisp design decision (tornado sacrifice, ghost replay, enrage phase) and an agent executed it fully in one pass. Clarity in, quality out.
