M16–M20 Features
  • M16 Round Variety: STANDARD / BLITZ (6 pennies, 700ms, 1.5× bugs) / ENDURANCE (14 pennies, 1.6s). Rolled per round. Badge in HUD, labeled in countdown.
  • M17 Bug AI: Dodge behavior — tap within 60px but not a hit triggers bug.dodge() (±30px perp offset over 400ms). Wave 5+ swarm event: 3 bugs simultaneously every 30s.
  • M18 Accessibility: Tap feedback rings (8→40px expanding white ring, 200ms fade). Color-blind mode (long-press mute 500ms): pennies → blue #44aaff, gold bugs → yellow #ffdd44. Landscape overlay with rotate icon.
  • M19 Performance: Off-screen culling (30px margin skips draw). FPS debug counter (?debug=1). gameState.debug tracking object.
  • M20 Void Events: 0.3% per-frame chance. Types: SLOW PENNY (5s, penny ages 5× slower), BUG FREEZE (3s, bugs at 30% speed), DOUBLE POINTS (next 3 claims at 2×). Notification text at seam. voidEvents.js system with getSlowFactor/getBugFreezeFactor/getVoidEventMultiplier.
⚡
Void Event System
SLOW PENNY

5 seconds. Active penny ages at 0.2× rate — dramatically extended lifetime. Glows expectantly in the void.

BUG FREEZE

3 seconds. All bugs move at 30% speed. Brief window to clean up the screen.

DOUBLE POINTS

Next 3 penny claims worth 2× points. Counter decrements on each claim. Seam text announces it.

🐛
Known Bugs at M20

MINORVoid event chance is per-frame at ~60fps, making it ~18% per second — higher than intended 0.3%/frame at 60fps (~0.3%). Actually correct but feels more frequent than expected.

MINORLong-press color-blind toggle in game.js increments muteLongPressTimer per-event per-frame, but may fire both mute-toggle and color-blind-toggle simultaneously.

MINORDOUBLE POINTS void event counter (doublePtsRemaining) is module-level state in voidEvents.js, not in gameState — won't reset correctly if resetVoidEvents is called mid-event.
