What Was Built — M1 to M5
  • M1 — Core Gameplay: Split-screen canvas, Penny entity (drop-in animation, alpha fade, copper draw), Bug entity (AI path, sine wobble, wing flicker), Particle pool (128 slots), Spawner (10 pennies, 40/40/20 zone distribution), Difficulty wave system, Collision resolution (bugs before pennies, no ghost hits), Title/Game/RoundEnd screens, Flash FX.
  • M2 — Audio: Web Audio API synth.js — playCoin (C5→E5), playBug (noise), playMiss (sawtooth), playWin (4-note arpeggio), playBeep (countdown). 3-2-1 countdown, mute toggle with localStorage, score scale animation on change, penny expiry flicker, fixed penny despawn round-end logic.
  • M3 — Juice: Particle bursts on penny collect and bug kill, screen flash on round end, seam divider line, audio polished and wired throughout. Score number bounce animation. Round end shows score comparison and bugs shot per player.
  • M4 — Best-of-3: roundWins[0/1] tracking, roundHistory per-round snapshots, Match End screen (src/screens/matchEnd.js) with series winner, per-round score breakdown, win dots. Series dots on round end screen (● ○). Correct round progression through best-of-3 arc. main.js wired with MATCH_END case.
  • M5 — Share + Void Depth: Share button on Match End (navigator.share → clipboard fallback). Share text includes scores and URL. src/fx/voidDepth.js — 60 semi-transparent drift particles (rgba 0.035 opacity), wrapped at canvas edges, registered at draw layer 0. initVoidDepth() called from main.js.

Known Bugs
Active Issues at M5

MINORRound end series dots show current wins after the round is recorded — but if the round is a tie, no win is awarded. A series can theoretically never reach 2 wins if every round ties (unlikely but possible).

MINORVoid depth particles initialize size on first draw call rather than on resize. If canvas resizes mid-session, particles may cluster in a sub-rectangle until wrapping corrects them.

MINORShare button hit area uses approximate Y coordinates relative to dot positions — may miss on very small or very large screens where seam is not at expected position.

COSMETICTitle screen pennyPulseAge still increments by ~16ms per draw call (not real dt). Slight drift at non-60fps displays.

COSMETICMute button visually appears in both P1 and P2 HUD halves — correct behavior but could confuse new players who don't realize either corner works.

Architecture Overview
Module Map

src/state.js — Single source of truth. gameState, freshRoundState, resetRound, resetMatch, SCREENS enum, roundHistory, roundWins.

src/main.js — RAF game loop. Screen state machine. Wires all screens. Calls initVoidDepth() at startup.

src/renderer.js — DPR-aware canvas. registerLayer(fn, priority). Layers drawn lowest→highest each frame. Seam divider drawn as a layer.

src/entities/penny.js — Penny class. Drop-in cubic ease, lifetime alpha fade, urgent flicker, hit test.

src/entities/bug.js — Bug class. Direction vector toward target half center, sine wobble, wing flicker, miss at canvas boundary.

src/entities/particle.js — 128-slot pool, spawnBurst(), gravity, alpha fade.

src/systems/spawner.js — 10 pennies/round, 1.2s interval, max 3 on screen, 40/40/20 zone distribution.

src/systems/difficulty.js — Bug wave escalation: speed 80→180px/s, interval 5s→2s, wave 3+ side spawns.

src/systems/collision.js — Bugs resolved before pennies, per-touch entity resolution.

src/systems/scoring.js — claimPenny, shootBug, missBug, isRoundOver.

src/fx/flash.js — Half/full screen color flash queue.

src/fx/voidDepth.js — 60 drift particles at layer 0 (M5).

src/audio/synth.js — Web Audio API synth. All sounds synthesized, zero audio files.

src/screens/title.js, game.js, roundEnd.js, matchEnd.js — Screen logic + draw layers.

Recommendations & Ideas
Technical Recommendations

Before M6: Refactor penny/bug scoring to support per-type point values cleanly — scoring.js currently uses fixed constants (PENNY_POINTS=1, BUG_KILL_POINTS=2) but M7 needs per-entity point lookups.

voidDepth.js should track canvas dimensions at init and re-initialize particles on resize rather than relying on wrap-around correction.

Consider moving round type selection (M16) to state.js — roundType field alongside chaosMode.

Design Ideas

Gold bugs + wheat pennies in the same round create a high-risk/high-reward moment — a player chasing a +5 gold bug while ignoring 3 pennies is a real strategic decision.

The void depth particles at 0.035 opacity are barely visible — intentionally so. Consider adding 3-5 slightly brighter copper floaters (0.08 opacity) that drift more slowly as an easter egg depth layer.

Share result text could include emoji for quick scanning: "🪙 Penny Panic: P1 7 – P2 12".
