M11–M15 Feature Summary
  • M11 Chaos Mode: 20 pennies, 2× bug speed, 5s lifetime. Both players tap simultaneously (<500ms) on match end to enter. CHAOS label in red at seam center. Chaos round always goes to match end when done.
  • M12 Tutorial: First-play overlay during countdown — penny icon (+1) and bug icon (+2, miss -1) with labels for each half. Fades after 4s or on tap. localStorage 'pp_played' gate.
  • M13 Audio: Two-oscillator coin chord (C5+E5), positional stereo panning ±0.3 by player half, lowpass filtered bug crunch (800Hz), sub-bass+sawtooth miss, 8-note match win fanfare, chaos entry cluster, combo ascending tones (E5/G5/C6).
  • M14 Seam Tension: CONTESTED text at seam on center-zone penny spawn. CLASH! red text burst when two players tap same penny simultaneously. Seam wins tracked per player. Round end shows "N seam wins".
  • M15 Stats: Match stats tab on match end (RESULTS/STATS toggle). Tracks pennies collected, bugs shot, seam wins, biggest combo per player. Career stats persisted to localStorage 'pp_career'. Title screen shows "N games · N pennies" per player.

Known Bugs at M15

Architecture Notes
Module Count & Structure

~20 source files across: entities (penny, bug, particle), systems (spawner, difficulty, collision, scoring), screens (title, game, roundEnd, matchEnd), fx (flash, voidDepth), audio (synth), and core (state, renderer, input, main).

The seam (canvasH/2) is consistently computed dynamically — zero hardcoded screen dimensions survive beyond constants.js reference values.

gameState is a true single source of truth. All systems read from and write to it. No hidden module-level state except UI timers (countdownActive, waitTimer) and the audio context (intentionally isolated).

Performance: particle pool is pre-allocated, no dynamic allocation per frame for particles. Entity arrays still use filter() on every frame — M19 will address this.
