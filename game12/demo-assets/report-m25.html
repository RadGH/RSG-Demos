Known Remaining Issues

MINORVoid event chance is 0.3%/frame — at 60fps this is ~17%/second, higher than it reads. Feels roughly correct but math is less elegant than a per-second trigger.

MINORLetter drop-in uses measureText('M') for character width — proportional in some fonts. Monospace fallback handles this correctly.

MINORCelebration particles from match winner are not clipped to winner half — they can overlap opponent area.

MINORdoublePtsRemaining in voidEvents.js is module-level state, not in gameState — resetVoidEvents() resets it but it could desync if called while decrement is in flight.

MINORSettings screen rows for P2 use mirrored Y detection but same X range (centerX ± halfWidth) — close enough for the centered layout but could theoretically miss edge row taps at extreme angles.

What Makes Penny Panic Special
The Void Is The Point

The emptiness of the void was the first design requirement. Two people sharing a phone. Nothing but pennies falling. The void depth particles, vignette, ambient hum — everything exists to make the emptiness feel intentional and beautiful rather than unfinished.

Zero Dependencies, Zero Assets

43 kB of pure JavaScript. No audio files, no images, no frameworks. The entire game — sounds, art, animations — is generated at runtime via Web Audio API and Canvas 2D. The copper penny is radial-gradient CSS. The bug is a dozen canvas paths. Everything is the void plus math.

Same Device, Different Worlds

P1 reads top to bottom. P2 reads bottom to top (rotated 180°). The seam at canvasH/2 is both dividing line and contested territory. CONTESTED text, CLASH!, seam wins, penny types that spawn near center — the split-screen mechanic has more depth than it first appears.
