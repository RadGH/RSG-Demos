AI Integration

In-game AI that reads the mod schemas + currently-loaded modules and generates new content on demand.

Content-generation AI — describe a character or spell, AI writes the JSONtodo
AI has read access to `/schemas/v1/` and the active module set. It validates against the schema before returning. Used for quick custom heroes, spells, companions, encounters.
Portrait-generation AI — reference photo + directional framestodo
Separate generator that follows the current SpriteCook flow: one reference image, then derived portrait/south/east/east_attack/east_spell/east_block/east_ko frames with style consistency.
Image review tool — whole-sprite prompt modetodo
Selecting the whole sprite in `/assets/image-review.html` applies one prompt to all connected views (portrait, south, east, east_attack, …). Reference image optional — leave blank to regenerate from scratch with the prompt.
Image review tool — archive / library / delete per imagetodo
Checkbox on each image: move to library (keeps as selectable alternate appearance, no longer linked to a Hero), archive, or delete. Supports cleanup after iteration.
Image review tool — "Unassigned" appearance categorytodo
Bottom section of Appearances listing entries not linked to any class but still selectable for custom characters in-game.
Image review tool — "Archived / Unattached" categorytodo
Catch-all for orphan images — shows what's on disk but not wired to anything, with delete / restore / assign actions so the pool can be cleaned or completed.
