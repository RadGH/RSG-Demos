Affix Audit & Survey

How to use: For each existing affix, choose what should happen (keep, retune, rework, or remove), adjust the min/max range, and add notes. At the bottom you can propose new affixes. When done, press Generate Prompt to produce a paste-back instruction Claude will use to implement your decisions.

Status legend: WIRED reads the affix in combat/stat code. PARTIAL stat exists but may not sum from equipment. ORPHAN rolled on items but never read anywhere — dead code.

Note: In the current build, character stats are computed in CombatScreen._memberToCombatant directly from base attrs + passives + item.armor + item.dmg. Affix values are NOT summed into character stats, except mana_regen which is read at regen time. So almost every affix below is effectively orphaned until we wire up a shared getEquipmentAffixBonuses() function.
