Accessibility · Statement

Built to be navigable for everyone.

Emberveil targets WCAG 2.1 Level AA. The site is keyboard-friendly, respects reduced motion, and ships sensible defaults for low-vision users. Where we fall short, we say so here.

Compliance target

Both the game (the React app at /play) and the marketing site (this page, news, tools, catalogs) target WCAG 2.1 Level AA. Compliance is verified per release through manual keyboard nav, screen-reader smoke tests, and contrast spot-checks.

Keyboard navigation

Every interactive element is reachable with Tab. Focus is visible at all times — the rune-circle cursor on the homepage is decorative only and does not hide focus rings.

  • Tab / Shift+Tab — move between focusable elements.
  • Enter / Space — activate the focused button or link.
  • ↑ ↓ ← → — within the class picker, tab list, and image-review filter groups.
  • Home / End — jump to the first / last item in a list group.
  • Esc — close any open lightbox or modal.

Reduced motion

If your OS reports prefers-reduced-motion: reduce, we automatically:

  • Disable parallax scrolling in the hero.
  • Hide drifting ember particles entirely.
  • Pause the combat-preview auto-advance.
  • Clamp every CSS transition to 0.01ms (effectively static).

The static fallback still conveys all content — only motion is removed.

Screen readers

Every chart on the homepage carries a textual aria-label describing the data. Form inputs have associated labels. Decorative icons are marked aria-hidden. We test with VoiceOver (Safari/iOS) and NVDA (Firefox/Windows). If a chart, button, or link reads ambiguously to your reader, please report it.

Color & contrast

The Ember accent (#e8612a) is used for decorative borders and glows only. All interactive text and links use a brighter accent variant (#ffa844, --accent-text) that meets WCAG AA contrast on the dark background.

Touch targets & mobile

Buttons, links, and tap surfaces are at least 44×44 CSS pixels on mobile breakpoints. The site has been tested on iPhone 14 Pro Safari (393×852) and renders without horizontal scroll.

Known limitations

  • The combat-preview auto-advance loop on the homepage runs while the page is visible. It pauses when you tab away or when reduced-motion is set, but cannot be paused independently of motion preferences via the UI yet.
  • The image-review tool's lightbox supports keyboard nav (Esc / arrows) but does not yet trap focus inside the modal — Tab will leak to the underlying page.
  • Pixel-art sprite zoom-in (in the asset gallery) does not yet have keyboard equivalents — only mouse hover.

Reporting issues

Found something that's hard to use? Please mail radleygh@gmail.com or use the contact page. We treat accessibility regressions as bugs, not enhancements — they go to the front of the queue.
