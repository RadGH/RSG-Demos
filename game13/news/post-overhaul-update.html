Cloud saves & telemetry — opt-in by default

M325 shipped Supabase-backed cloud saves; M333 layered telemetry + GA4 on top. Both are strictly gated on the privacy banner: nothing fires until the user clicks accept. localStorage.rsg_consent_v1 is the gate; the GA snippet, telemetry events client, and saves writer all check it on every flush. Decline once and we never load gtag.js at all — no fingerprinting, no shadow tracking.

Each saved run captures run_stats: duration, kills, treasure found, companions deployed, talent picks, final hero level. This feeds the new Achievements panel + the per-property GA dashboards (G-XP7B7KVNY8 for staging, G-5H26ZXKBCL for emberveil.radgh.com).

NPC + class art

The redesign pipeline reached full maturity through M260+. Kaela Thorne and Marek Greel received seven-pose SpriteCook regens. Every regen now pulls verbatim from public/data/art_direction/<id>.json — the canonical source — so the prompts cannot drift. The M349 pyromancer regen taught us that ad-hoc MCP calls must obey the same image-policy rule as bulk runs: pixel mode, transparent background, download from pixel_url not raw_url. That contract is now in public/docs/image-policy.md with a reinforcement note.

By the numbers

Act win rates — pre & post M350 balance pass

Pre-pass, Comp B (4 heroes + companion) was unwinnable in Act 5 (0.6%) and trivial in Act 6 (84.9%). Post-pass: Act 5 jumped to 53.4% and Act 6 dropped to 66.6%. Acts 1–4 deliberately untouched — they were already inside the target band.

Damage share — Comp B heroes, Acts 1-6 averaged

Mage leads at 24% of party damage, followed closely by Warrior (22%), Fighter (18%), Ranger (18%), and the companion (18%). The companion's contribution is meaningful but never dominant — the comp is still hero-driven.

Status effect frequency in Monte Carlo runs

Burn (35%) and Dazed (22%) dominate after M341 converted several stun skills to dazed. Stun (15%) is now reserved for crowd-control skills with longer cooldowns. Poison (10%) is rare by design — it's the long-game DoT, not the workhorse.

UI unification (M312+)

  • Settings — grid layout, category tabs, debug flags pulled out into a separate Debug panel.
  • Title screen — difficulty picker is the first option; Hard mode disables fast-travel teleport.
  • Save / Load — date-stamped slots with preview stat panels; party portraits inline.
  • Party overview — portrait grid with scroll sync to equipment + skill trees.
  • Hire Custom — matches Create Hero; appearance grid + estimated stats.

Tooling — Image Review V2 + live tools index

M351 launched Image Review V2, replacing the older Image Review and Character Redesign pages. It supports a Pending / Current / History mode toggle, multi-select with copy as list for approval handoff, and an image-left/sidebar-right lightbox with old/new comparison and prompts in the side panel. Approval is a two-step relay: select sprites in the page, paste the list back to Claude, who runs scripts/approve-image-review-v2.cjs. The first batch (m350 — pyromancer east_spell) is already in history.

Tools page is now live data. build-tools-list.cjs scans public/assets/*.html on every release and rebuilds tools.json. New tools surface immediately with a placeholder description prompting curation.

Looking ahead

  • Branching dialog & quests — multi-outcome conversations with conditional combat triggers.
  • Passive skill trees — talent picks persisted across level-ups, talent reset items.
  • Polish pass — animation timing, sound feedback, micro-juiciness.
  • Performance audit — combat scene rendering on low-end devices (M357 wishlist).

The fast-travel overhaul was about navigation. The post-overhaul arc was about foundation: saves, telemetry, art pipeline, balance, tooling. With foundation set, the next phase can finally be content — story branches, companion arcs, scaling difficulty that feels adaptive instead of scripted.
