Executive Summary

What Is Emberveil?

Emberveil is a high-fantasy party RPG that runs entirely in the browser — no install, no app store. A lone hero grows into a party of 8 as they chase the Emberveil Corruption across four wildly different realms: medieval goblin kingdoms, volcanic wastelands, the Shattered Hell, and the Cosmic Void. Combat is cinematic and automatic, driven by smart AI with deep customization available between rounds. Every piece of equipment is procedurally generated. Every encounter tells a story.

Feature Deep Dive

Core Feature Matrix

Cinematic Auto-Combat

Characters act every 0.5 seconds in turn order. Each unit switches stance (attack/spell/death), moves on-screen, and triggers particle effects. Special abilities add suspenseful delays.

AoE Group Combat

Enemies are grouped in packs of 2-4 (max 12 total). Fireball hits a pack. Blizzard hits all packs. Even melee hits 1-3 adjacent targets. Group position matters.

Procedural Equipment

Every item is generated from base type + rarity (Normal/Magic/Rare/Legendary) + quality (Low → Exotic) + affix pool. Billions of possible items. 10 equipment slots per character.

Party of 8

4 heroes + 4 companions. Start alone, hire from taverns, rescue in quests, summon via skills. Companions include war dogs, dragon whelplings, raised skeletons, and summoned wolves.

Dialog + Skill Checks

Encounters use portrait-left/NPC-right dialog with scrolling text. Choose persuasion, stealth, or lore to avoid combat or gain advantage. Choices persist as story flags.

Talent Trees

Each class has 4 skills unlocked at levels 1/5/10/15. Each skill has 2-3 upgrade talents that change how it works — add Burn, extend duration, widen AoE, or reduce cooldown.

Node Map Travel

Linear maps with 3-8 nodes and branching paths. Fast travel to any visited node. Zone map shows multiple maps per act. Cinematic party movement between nodes.

OpenAI Backgrounds

Each region has AI-generated scenic backgrounds. Goblin frontier forests, volcanic ash clouds, hellfire landscapes, and cosmic void vistas — all generated via OpenAI API.

Web Audio Music

Dynamic ambient soundtrack that shifts with context — tavern folk tunes, tense combat percussion, atmospheric void ambience. Procedurally layered using Web Audio API.

Fully Responsive

iPhone 14 Pro primary (393×852). Portrait lock on mobile. Mouse, touch, keyboard, and gamepad all fully supported with on-screen button indicators for controller.

Quest Journal

Main quests and side quests tracked with objectives, map markers, and rewards. Completing side quests unlocks powerful named companions and unique legendary items.

New Game+

After completing Act 4, start over at Normal → Hard → Nightmare with higher level enemies. Story choices carry minor narrative acknowledgements across playthroughs.

Technical Architecture

Tech Stack

Canvas Layer — Game World
HTML5 Canvas 2D for all game visuals: backgrounds, character sprites, combat animations, particle systems, map rendering, cinematic sequences.
HTML/CSS Layer — UI
All menus, panels, dialogs, tooltips, party displays, inventory grids, skill trees, and HUD elements. Positioned absolutely over canvas. SVG custom icons throughout.
Web Audio API — Sound
Procedurally generated ambient music using oscillators, filters, and gain nodes. Sampled SFX for combat. Context-aware music layers — town, combat, boss fight, cinematic.
Art Generation — PixelLab + OpenAI
PixelLab MCP for all character art (5 stances × 14 classes + companions + enemies). OpenAI API for region backgrounds. SVG for UI icons and decorative elements.
Data Layer — localStorage + JS Modules
Vite + Vanilla JS modules. No framework overhead. localStorage save system with 3 slots, full game state serialization. JSON content files for items, enemies, encounters, and dialog.

Risk Assessment

Risks & Mitigations

RiskSeverityMitigation
Mobile canvas performance
12 enemies + animations + particles on mobile Safari could drop below 30fps		 High Off-screen sprite culling, sprite sheet atlasing, requestAnimationFrame budget cap (16ms), reduce particle count on low-end detection, canvas resolution scaling on DPR >2.
PixelLab art volume
5 stances × 14+ classes + enemies + companions = 200+ generations before Act 2		 Medium SVG placeholder stances for all characters in M1–M5. Generate final art in parallel batches. Prioritize player-visible characters first. Track credit usage per milestone.
Save format compatibility
50 milestones means the data schema will evolve; old saves may break		 Medium Version field in every save slot from day one. Migration functions run on load to upgrade old schemas. Never remove a field — mark deprecated and ignore.
Skill system complexity
14 classes × 4 skills × 3 talents each = 168 effects to balance and test		 Medium Build a shared effect engine with composable primitives (damage, DoT, AoE, buff, debuff). New skills are just JSON configs — no new code per skill. Balance pass at each act boundary.
OpenAI API costs
Background generation at scale (16+ zones) could accumulate significant API costs		 Low Generate once and cache to disk. Never regenerate existing assets. Use 1024×1024 standard quality (not HD) for backgrounds. Store in public/images/ as permanent assets.
Creative Possibilities

Suggestions & Ideas

  • Daily Challenge Mode: A seed-generated map shared by URL hash. All players face the same encounters and bosses. Leaderboard by fastest clear or highest score. Zero implementation cost once core map system exists.
  • Companion Personalities: Each hireable companion has a personality trait (Loyal, Greedy, Brave, Cowardly) that affects their dialog in encounters and sometimes overrides their combat AI in interesting ways.
  • Void-Touched Items (New Game+ exclusive): Legendary items that have a fourth affix slot which can only roll Void-type modifiers. Visually distinct with purple corruption particle effects.
  • Environmental Hazard Nodes: Act 2 volcanic vents, Act 3 brimstone rivers — instead of combat, navigate a mini-game using party stats to avoid damage. Breaks up encounter monotony.
  • Hidden Bosses: Each act has one optional boss only accessible by completing all side quests in that act. These drop Legendary gear and unique companions not obtainable any other way.
  • Companion Quest Lines: Each named companion has a 3-step side quest chain. Completing their chain unlocks a unique ability or stance they gain permanently. Rewards player investment in specific companions.
  • Procedural Side Events: Between nodes, random flavor events: "A merchant's cart has overturned — help them?" Small gold rewards, occasional rare item. Keeps travel nodes from feeling empty.
  • Photo Mode: During dialog stances or victory screens, a screenshot button captures the canvas + UI as a PNG. Let players save and share their party compositions and moments.
Development Cautions

Watch Points

Test on Real Mobile Devices Early

Safari on iPhone has quirks: Web Audio context requires user gesture to start; touch events behave differently than mouse events; canvas DPR needs explicit handling. Test M2 on iPhone before assuming desktop behavior translates.

Combat AI Must Feel Smart Immediately

Auto-play combat is only fun if the AI makes interesting choices. A Pyromancer that casts Flame Lance on a single target while ignoring a 6-enemy group ruins immersion. Invest in the AI priority system in M5-M7 before adding more skills.

Tooltip Hover System is Critical Infrastructure

The spec requires every item, skill, and stat to show a detailed tooltip with formulas shown. Build this as a shared component early (M6). It will be used hundreds of times. A late-stage tooltip refactor at M30 would be very painful.

localStorage has a 5-10MB limit per origin

Full game state including item inventories, story flags, and quest tracking can grow large. Consider compressing save data with a simple JSON diff scheme. Budget ~500KB per save slot to stay safe.
